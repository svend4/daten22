## Введение: Проблема Информационной Пирамиды

### Четыре Уровня Информационной Пирамиды

```
    ▲  УРОВЕНЬ 1: ВЕРШИНА
   ███  Один простой вопрос
  █████  Пример: "Города Европы"
 ███████
         ↕ РАЗРЫВ (нужен алгоритм)

██      УРОВЕНЬ 2: СТРУКТУРИРОВАННЫЙ СКЕЛЕТ
███     Детальный вопросник (сотни пунктов)
████    Организованный по архетипам
█████
         ↕

████████ УРОВЕНЬ 3: ОТФИЛЬТРОВАННЫЕ ДАННЫЕ
████████ Структурированные ответы
████████ Информация из Level 4, организованная по шаблону
████████
         ↕

████████ УРОВЕНЬ 4: ОСНОВАНИЕ
████████ Огромный массив неструктурированных данных
████████ Миллионы документов, статей, новостей
████████ Хаотичная информация
```

**Цель алгоритма:** Заполнить разрыв между Уровнем 1 и Уровнем 2.

---

## Философия Алгоритма

### Принцип Фрактальной Декомпозиции

Любой вопрос можно разложить на 16 под-вопросов (по количеству архетипов), каждый из которых можно снова разложить на 16, и так далее.

```
Вопрос
├── Архетип 1 (MSEO)
│   ├── Подвопрос 1.1
│   ├── Подвопрос 1.2
│   └── ...
├── Архетип 2 (MSEF)
│   ├── Подвопрос 2.1
│   └── ...
├── ...
└── Архетип 16 (ADCF)
    ├── Подвопрос 16.1
    └── ...
```

**Глубина декомпозиции:** 3-4 уровня (16 → 256 → 4096 → 65536 вопросов)

**Практическая глубина:** 2-3 уровня (достаточно для большинства тем)

## АЛГОРИТМ: Пошаговый Процесс

### ШАГ 1: Парсинг Входного Запроса

**Вход:** Строка запроса (например, "Города Европы")

**Действия:**
1. Извлечь **ключевые сущности** (существительные)
2. Извлечь **контекст** (географический, временной, тематический)
3. Определить **домен знаний** (география, урбанистика, история и т.д.)
4. Определить **уровень детализации** (обзорный / детальный)

**Пример:**

```python
Запрос: "Города Европы"

Парсинг:
{
  "entities": ["город", "Европа"],
  "context": {
    "geography": "Европа",
    "domain": ["урбанистика", "география", "социология"],
    "time": "современность (по умолчанию)"
  },
  "detail_level": "обзорный"
}
```

## Формализация Алгоритма

### Псевдокод

```python
def expand_query(query: str, depth: int = 2, max_questions: int = 1000) -> QuestionTree:
    """
    Расширяет простой запрос в структурированный вопросник.

    Args:
        query: Входной запрос (например, "Города Европы")
        depth: Глубина декомпозиции (1-4)
        max_questions: Максимальное количество вопросов

    Returns:
        Иерархическое дерево вопросов
    """

    # ШАГ 1: Парсинг
    parsed = parse_query(query)
    entities = parsed['entities']
    domain = parsed['domain']
    context = parsed['context']

    # ШАГ 2: Определение релевантности архетипов
    archetypes = get_all_archetypes()  # 16 архетипов
    relevance_scores = {}

    for archetype in archetypes:
        relevance_scores[archetype] = calculate_relevance(
            archetype, entities, domain, context
        )

    # Сортировка по релевантности
    sorted_archetypes = sort_by_relevance(relevance_scores)

    # ШАГ 3: Генерация базовых вопросов (уровень 1)
    question_tree = QuestionTree(root=query)

    for archetype in sorted_archetypes:
        if relevance_scores[archetype] >= THRESHOLD:
            base_question = generate_base_question(archetype, query)
            question_tree.add_child(base_question)

    # ШАГ 4: Рекурсивная декомпозиция
    for base_question in question_tree.children:
        if depth > 1:
            sub_questions = decompose_question(
                base_question,
                archetypes,
                depth - 1
            )
            base_question.add_children(sub_questions)

    # ШАГ 5: Добавление взаимодействий
    if depth >= 2:
        interaction_questions = generate_interaction_questions(
            sorted_archetypes,
            query
        )
        question_tree.add_section("interactions", interaction_questions)

    # ШАГ 6: Приоритизация и обрезка
    if len(question_tree) > max_questions:
        question_tree = prioritize_and_prune(question_tree, max_questions)

    return question_tree


def generate_base_question(archetype: Archetype, query: str) -> Question:
    """
    Генерирует базовый вопрос для архетипа.
    """
    template = QUESTION_TEMPLATES[archetype.code]
    question_text = template.format(topic=query)

    return Question(
        id=f"{archetype.code}_base",
        text=question_text,
        archetype=archetype,
        priority=calculate_priority(archetype, query)
    )


def decompose_question(question: Question, archetypes: List[Archetype], depth: int) -> List[Question]:
    """
    Рекурсивно декомпозирует вопрос на подвопросы.
    """
    if depth == 0:
        return []

    sub_questions = []

    # Вариант 1: Декомпозиция по аспектам архетипа
    aspects = get_archetype_aspects(question.archetype)
    for aspect in aspects:
        sub_q = Question(
            text=f"{question.text} - {aspect.name}",
            parent=question,
            depth=question.depth + 1
        )
        sub_questions.append(sub_q)

        # Рекурсия
        if depth > 1:
            deeper = decompose_question(sub_q, archetypes, depth - 1)
            sub_q.add_children(deeper)

    return sub_questions


def generate_interaction_questions(archetypes: List[Archetype], query: str) -> List[Question]:
    """
    Генерирует вопросы о взаимодействиях между архетипами.
    """
    interaction_questions = []

    # Берем топ-10 наиболее релевантных архетипов
    top_archetypes = archetypes[:10]

    for i, arch1 in enumerate(top_archetypes):
        for arch2 in top_archetypes[i+1:]:
            interaction_type = INTERACTION_MATRIX[arch1.code][arch2.code]

            if interaction_type in HIGH_PRIORITY_INTERACTIONS:
                q = Question(
                    text=f"Как {arch1.name} взаимодействует с {arch2.name} в контексте {query}?",
                    archetype_pair=(arch1, arch2),
                    interaction_type=interaction_type
                )
                interaction_questions.append(q)

    return interaction_questions
```

## Шаблоны Вопросов для Каждого Архетипа

### MSEO (Кристалл) - Элементарные Материалы
```
- Какие базовые материалы используются в {topic}?
- Из каких элементов состоит {topic}?
- Какие физические свойства важны для {topic}?
```

### MSEF (Песок) - Сыпучие Элементы
```
- Какие неупорядоченные материалы присутствуют в {topic}?
- Какие гранулированные вещества связаны с {topic}?
```

### MSCO (Здание) - Архитектура
```
- Какие структуры и сооружения составляют {topic}?
- Какая архитектура характерна для {topic}?
- Какие типы зданий встречаются в {topic}?
```

### MSCF (Лес) - Природный Ландшафт
```
- Какая природная среда окружает {topic}?
- Как ландшафт интегрирован в {topic}?

```

## Заключение: Мощность Алгоритма

### Достижения:

1. **Превращает 1 вопрос в 500-3000 структурированных подвопросов**
2. **Гарантирует полноту охвата** (все аспекты через 16 архетипов)
3. **Адаптивность к любой теме** (меняется только релевантность архетипов)
4. **Иерархическая организация** (легко навигировать)
5. **Масштабируемость** (можно контролировать глубину)

### Применение:

- **Написание энциклопедий** (структура статей)
- **Исследовательские проекты** (полнота анализа)
- **Образовательные программы** (учебные планы)
- **PseudoRAG система** (шаблон для поиска данных)
